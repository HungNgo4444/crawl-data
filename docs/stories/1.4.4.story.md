## Technical Specifications

### Database Schema

#### Raw Articles Table
```sql
CREATE TABLE raw_articles (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    url TEXT NOT NULL,
    url_hash VARCHAR(64) UNIQUE NOT NULL,
    domain_id UUID REFERENCES domains(id),
    
    -- Article Content
    title TEXT,
    content TEXT,
    publish_date TIMESTAMP,
    author JSONB DEFAULT '[]',
    category VARCHAR(255),
    url_image TEXT,
    
    -- Processing Metadata
    extraction_model VARCHAR(50) DEFAULT 'newspaper4k-extractor',
    processing_time_ms INTEGER,
    content_quality_score DECIMAL(3,2) DEFAULT 0.0,
    extraction_errors JSONB DEFAULT '{}',
    
    -- Timestamps
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    
    -- Indexes
    UNIQUE(url_hash, domain_id)
);

-- Performance indexes
CREATE INDEX idx_raw_articles_domain_created ON raw_articles(domain_id, created_at);
CREATE INDEX idx_raw_articles_publish_date ON raw_articles(publish_date);
CREATE INDEX idx_raw_articles_url_hash ON raw_articles(url_hash);
CREATE INDEX idx_raw_articles_category ON raw_articles(category);
CREATE INDEX idx_raw_articles_extraction_model ON raw_articles(extraction_model);
```

#### URL Tracking Status Updates
```sql
-- Update URL tracking status during processing
ALTER TABLE url_tracking ADD COLUMN IF NOT EXISTS processed_by VARCHAR(50);
ALTER TABLE url_tracking ADD COLUMN IF NOT EXISTS processing_started_at TIMESTAMP;
ALTER TABLE url_tracking ADD COLUMN IF NOT EXISTS processing_completed_at TIMESTAMP;
```

### Container Configuration
```dockerfile
# Multi-stage Dockerfile
FROM python:3.11-slim-bullseye as builder

# Install system dependencies for newspaper4k
RUN apt-get update && apt-get install -y \
    gcc g++ \
    libxml2-dev libxslt1-dev \
    libjpeg62-turbo-dev zlib1g-dev \
    curl \
    && rm -rf /var/lib/apt/lists/*

# Install Python dependencies
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Production stage
FROM python:3.11-slim-bullseye

# Copy installed packages from builder
COPY --from=builder /usr/local/lib/python3.11/site-packages /usr/local/lib/python3.11/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin

# Create app user
RUN useradd --create-home --shell /bin/bash app
WORKDIR /app

# Copy application code
COPY src/ ./src/
COPY scripts/ ./scripts/

# Set permissions
RUN chown -R app:app /app
USER app

# Environment variables
ENV PYTHONPATH=/app
ENV DATABASE_URL=""
ENV BATCH_SIZE=50
ENV CONCURRENT_WORKERS=5
ENV LOG_LEVEL=INFO

# Health check
HEALTHCHECK --interval=30s --timeout=10s --start-period=60s \
    CMD python scripts/health_check.py

# Start application
CMD ["python", "-m", "src.main"]
```

### Processing Workflow
```
1. Container Startup
   ├── Load configuration từ environment variables
   ├── Establish database connection
   ├── Validate database schema
   └── Start processing workers

2. URL Processing Loop
   ├── Query url_tracking table for 'discovered' URLs
   ├── Mark URLs as 'processing' với batch lock
   ├── Process URLs using newspaper4k extractor
   └── Update processing status

3. Article Data Extraction
   ├── Extract article content using newspaper4k
   ├── Validate extracted data quality
   ├── Clean và normalize Vietnamese content
   └── Generate quality score

4. Data Storage
   ├── Insert article data vào raw_articles table
   ├── Update url_tracking status to 'completed'/'failed'
   ├── Log processing statistics
   └── Continue với next batch
```

### Input/Output Format

**URL Tracking Query:**
```sql
-- Select URLs for processing
SELECT id, original_url, domain_id, metadata 
FROM url_tracking 
WHERE status = 'discovered' 
  AND processing_attempts < 3
  AND (processing_started_at IS NULL 
       OR processing_started_at < NOW() - INTERVAL '1 hour')
ORDER BY created_at ASC 
LIMIT 50;
```
