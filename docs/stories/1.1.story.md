### Data Models

**Key tables to implement based on architecture specifications:**

#### DomainConfiguration Table
[Source: docs/architecture.md#data-models-and-schema-changes]
- **Purpose**: Store crawlable domain configurations và metadata cho 200+ domains management
- **Primary Key**: id (UUID) - Unique domain configuration identifier  
- **Core Fields**:
  - domain_name: VARCHAR(255) - Domain name (e.g., "vnexpress.net")
  - base_url: TEXT - Root URL cho crawling operations
  - status: ENUM (ACTIVE, INACTIVE, ANALYZING, FAILED)
  - crawl_frequency_hours: INTEGER - How often to crawl this domain
  - last_analyzed: TIMESTAMP - Last GWEN-3 analysis completion time
  - next_analysis_scheduled: TIMESTAMP - Next scheduled GWEN-3 analysis
  - success_rate_24h: DECIMAL(3,2) - Recent crawling success percentage
  - created_by_user: VARCHAR(100) - Admin user who added domain
  - created_at: TIMESTAMP - Domain addition timestamp
  - updated_at: TIMESTAMP - Last configuration update

#### DomainParsingTemplate Table  
[Source: docs/architecture.md#data-models-and-schema-changes]
- **Purpose**: Store GWEN-3 generated parsing rules cho each domain
- **Primary Key**: id (UUID) - Unique template identifier
- **Core Fields**:
  - domain_config_id: UUID - Foreign key to DomainConfiguration
  - template_data: JSONB - GWEN-3 analysis results và parsing rules
  - template_version: INTEGER - Version number cho template updates
  - confidence_score: DECIMAL(3,2) - GWEN-3 confidence trong parsing accuracy
  - structure_hash: VARCHAR(64) - Page structure fingerprint cho change detection
  - is_active: BOOLEAN - Whether this template is currently in use
  - created_by_analysis_run: UUID - Which GWEN-3 analysis run created this
  - expires_at: TIMESTAMP - When template should be refreshed
  - performance_metrics: JSONB - Success rates, extraction accuracy stats

#### DomainAnalysisQueue Table
[Source: docs/architecture.md#data-models-and-schema-changes]  
- **Purpose**: Manage daily GWEN-3 analysis scheduling cho 200+ domains
- **Primary Key**: id (UUID) - Unique queue entry identifier  
- **Core Fields**:
  - domain_config_id: UUID - Domain to analyze
  - scheduled_time: TIMESTAMP - When analysis should run
  - status: ENUM (PENDING, RUNNING, COMPLETED, FAILED, RETRYING)
  - analysis_started_at: TIMESTAMP - Actual analysis start time
  - analysis_duration_seconds: INTEGER - How long analysis took
  - gwen3_model_version: VARCHAR(50) - GWEN-3 model version used
  - error_message: TEXT - Error details if analysis failed
  - retry_count: INTEGER - Number of retry attempts
  - priority: INTEGER - Analysis priority (1-10, higher = more urgent)
